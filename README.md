# DevNotes (v1.1.4)

> ⚙️ **Статус проекта**: активная разработка  
> Проект находится в стадии активного развития. Новые функции, исправления и улучшения добавляются регулярно.

**DevNotes** — это open-source веб-приложение для ведения технического блога, 
разработанное на современном стеке Spring Boot. Проект предлагает готовую к использованию 
систему публикации статей с поддержкой администрирования, авторизации и загрузки медиафайлов.

Основная цель DevNotes — продемонстрировать простое, легко расширяемое и полностью автономное 
решение для создания блога без зависимости от внешних платформ или SaaS-сервисов.

---

## 🔧 Ключевые возможности

- Публикация, редактирование и удаление постов
- Административная панель с контролем доступа
- Защита маршрутов через Spring Security
- Авторизация администратора по логину и паролю
- Загрузка изображений с автоматическим сохранением на сервер
- Встроенный WYSIWYG-редактор TinyMCE для удобного форматирования текста
- Отображение списка всех постов и детальной страницы статьи
- Шаблонизация интерфейса с помощью Thymeleaf
- Централизованная обработка ошибок и исключений
- Профили Spring — раздельная конфигурация для разработки и продакшена

---

## 🛠 Технологический стек

### Backend
- **Java 21+**
- **Spring Boot** — основа приложения
- **Spring MVC** — обработка HTTP-запросов
- **Spring Security** — управление аутентификацией и авторизацией
- **Spring Data JPA + Hibernate** — работа с базой данных
- **PostgreSQL** — реляционная СУБД (продакшен)
- **H2 Database** — встроенная БД для разработки
- **Maven** — управление зависимостями и сборка проекта
- **CI/CD** — автоматический деплой при пуше в GitHub ветку main.

### Frontend
- **Thymeleaf** — шаблонизатор
- **HTML5 / CSS** — базовая разметка и стили
- **TinyMCE** — редактор контента

## 🌐 Доступные эндпоинты

| URL                       | Описание                                           | Доступ                |
|---------------------------|----------------------------------------------------|------------------------|
| `/`                       | Главная страница                                   | Все пользователи       |
| `/posts`                  | Список всех опубликованных постов                 | Все пользователи       |
| `/posts/{id}`             | Страница конкретной статьи                        | Все пользователи       |
| `/login`                  | Форма входа                                        | Все пользователи       |
| `/admin`                  | Административная панель                           | Только администратор   |
| `/posts/new`              | Форма создания нового поста                       | Только администратор   |
| `POST /posts`             | Сохранение нового поста                           | Только администратор   |
| `/posts/edit/{id}`        | Форма редактирования поста                        | Только администратор   |
| `POST /posts/update/{id}` | Обновление существующего поста                    | Только администратор   |
| `POST /posts/delete/{id}` | Удаление поста                                     | Только администратор   |
| `POST /images/upload`     | Загрузка изображения                               | Только администратор   |
| `/uploads/{filename}`     | Получение загруженного изображения                 | Все пользователи       |

> Все защищённые маршруты требуют успешной аутентификации администратора.

## 🚀 Установка и запуск

### 1. Предварительные требования

- **Java 21** или новее
- **Maven**
- **PostgreSQL** (только для продакшена)

---

### 2. Клонирование репозитория

```bash
git clone https://github.com/javer-63/DevNotes
cd DevNotes
```

---

### 3. Настройка конфигурации

Проект использует **профили Spring** для разделения настроек разработки и продакшена.

#### 🔧 Режим разработки (dev)

Для разработки используется встроенная H2 база данных — PostgreSQL не требуется.

Запустите приложение с этим профилем одним из способов:

**Через сборку JAR:**
```bash
mvn clean package
java -jar target/devnotes.jar --spring.profiles.active=dev
```

**Через Maven:**
```bash
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

> В режиме **dev** доступны SQL-логи в консоли и автоматическое создание таблиц (ddl-auto: update)

#### 🚀 Режим продакшена (prod)

Для продакшена требуется PostgreSQL.

1. Создайте базу данных:
```sql
CREATE DATABASE devnotes;
```

2. Настройте переменные окружения:
```bash
export DB_URL=jdbc:postgresql://localhost:5432/devnotes
export DB_USERNAME=your_db_username
export DB_PASSWORD=your_db_password
export ADMIN_USERNAME=your_admin_login
export ADMIN_PASSWORD=your_admin_password
```

3. Запустите приложение:
```bash
mvn spring-boot:run -Dspring-boot.run.profiles=prod
```

Или через JAR:
```bash
mvn clean package
java -jar target/devnotes.jar --spring.profiles.active=prod
```

> ⚠️ В продакшене используется `ddl-auto: validate` — таблицы должны быть созданы заранее.

---

### 4. Первый вход

После запуска откройте в браузере:

- Главная страница: http://localhost:8080
- Вход в админку: http://localhost:8080/login

Используйте логин и пароль, указанные в переменных окружения `ADMIN_USERNAME` и `ADMIN_PASSWORD`.

## 📝 Логирование

DevNotes использует встроенную поддержку Logback через Spring Boot для централизованного и гибкого логирования событий приложения. 
Все логи записываются как в консоль, так и в файл app.log, расположенный в корне проекта. Формат сообщений:

```log
2026-05-14 12:30:45 INFO PostController: Пост с URL xxx не найден
```

Основной файл: **app.log** — текущий активный лог.
Ротация логов:
* При достижении размера 10 МБ файл архивируется.
* Архивы именуются по шаблону: `app.2026-05-14.0.log.gz`, `app.2026-05-14.1.log.gz` и т.д.
* Хранятся логи за последние 7 дней.
* Общий объём всех архивных файлов ограничен 500 МБ — старые файлы автоматически удаляются при превышении лимита.

💡 Это позволяет эффективно использовать дисковое пространство даже на серверах с ограниченным объёмом памяти.
